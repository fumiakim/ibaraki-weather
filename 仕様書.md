# 仕様書

**プロジェクト名**: 雨雲ナウキャスト & 7日間天気予報ビューアー
**バージョン**: 1.0.0
**最終更新**: 2026-05-09

---

## 1. 概要

気象庁 (JMA) が公開する `bosai` 系オープンエンドポイントを直接利用し、複数地点の雨雲ナウキャスト、7日間天気予報、当日の気温推移を統合表示する単一ページアプリケーション。
ビルドツールやサーバ側コードは不要で、`index.html` をブラウザで開くだけで動作する。

## 2. 動作環境

| 項目 | 要件 |
| --- | --- |
| ブラウザ | Chrome / Edge / Safari / Firefox の現行版（Promise.allSettled, fetch, ES2020 構文必須） |
| ネットワーク | `https://www.jma.go.jp` および `https://cyberjapandata.gsi.go.jp` への HTTPS 接続 |
| 画面サイズ | 横幅 900px 以上を推奨（900px 未満は単カラムレイアウトに自動切替） |

## 4. 機能仕様

### 4.1 表示地点

| 地点名 | 緯度 | 経度 | ズーム | JMA office code | 一次細分 | 気温観測点 |
| --- | --- | --- | --- | --- | --- | --- |
| つくば | 36.075534 | 140.092506 | 13 | 080000 (茨城県) | 南部 | 水戸 |
| 柳津 | 37.511700 | 139.715800 | 13 | 070000 (福島県) | 会津 | 若松 |
| 姫路 | 34.815300 | 134.685400 | 13 | 280000 (兵庫県) | 播磨南西部 | 姫路 |
| 名古屋 | 35.181400 | 136.906500 | 13 | 230000 (愛知県) | 西部 | 名古屋 |

地点はソース内 `LOCATIONS` 配列で定義。追加・変更は当該配列の編集のみで完結する。

デフォルト表示地点は `LOCATIONS[0]`（つくば）。

### 4.2 雨雲ナウキャスト

- **データ種別**: 高解像度降水ナウキャスト (`hrpns`)
- **時刻範囲**: 過去1時間（実況, N1）＋ 1時間先まで（予報, N2）の 5 分刻み
- **最大フレーム数**: 24（過去12 + 予報12）
- **タイル仕様**:
  - URL: `https://www.jma.go.jp/bosai/jmatile/data/nowc/{basetime}/none/{validtime}/surf/hrpns/{z}/{x}/{y}.png`
  - ネイティブ最大ズーム: 10
  - 不透明度: 0.7
- **時刻一覧**:
  - 実況: `https://www.jma.go.jp/bosai/jmatile/data/nowc/targetTimes_N1.json`
  - 予報: `https://www.jma.go.jp/bosai/jmatile/data/nowc/targetTimes_N2.json`
- **N2 採用ロジック**: 最新の `basetime` を持つレコード群のみ採用し、`validtime > basetime` の条件で予報フレームを抽出

### 4.3 7日間天気予報

- **データ**: `https://www.jma.go.jp/bosai/forecast/data/forecast/{officeCode}.json`
- **3日先まで** (`data[0]`):
  - `timeSeries[0]`: 一次細分別の天気コード/天気文/風（`weatherCodes`, `weathers`, `winds`）
  - `timeSeries[1]`: 6時間ごとの降水確率（`pops`）
  - `timeSeries[2]`: 観測点別の朝/昼の最低/最高気温（`temps`）
- **4〜7日目** (`data[1]`):
  - `timeSeries[0]`: 日次の天気コード・降水確率・信頼度
  - `timeSeries[1]`: 観測点別の最低/最高気温（中央値・上限・下限）
- **細分の自動選択**: 地点の `subAreaMatch` 文字列で部分一致検索。見つからなければ index 0
- **気温観測点の自動選択**: 地点の `tempPointMatch` 文字列で部分一致。見つからなければ index 0
- **天気アイコン**: `https://www.jma.go.jp/bosai/forecast/img/{file}.svg`（`telops.json` から動的にマッピング、フェッチ失敗時は内蔵フォールバック表を使用）
- **日付集計**: ISO8601 タイムスタンプを Asia/Tokyo に変換し `YYYY-MM-DD` キーで日単位に集約。並べ替え後先頭 7 日分を表示

### 4.4 当日の気温推移（AMEDAS）

- **観測点表**: `https://www.jma.go.jp/bosai/amedas/const/amedastable.json`（起動時1回フェッチ＋プロセス内キャッシュ）
- **観測点選定アルゴリズム**:
  1. 地点の (lat, lon) と各観測点の (lat°, lat分 → degrees) との距離を経度差を `cos(lat)` で補正して二乗距離計算
  2. 距離昇順上位 5 候補
  3. 各候補について当日のデータを取得
  4. 気温データ (`temp[0]`) が存在した最初の候補を採用
  5. 5 候補すべてで気温が無い場合は「取得不可」表示
- **データ**: `https://www.jma.go.jp/bosai/amedas/data/point/{code}/{YYYYMMDD}_{HH}.json`
  - 3時間ブロック × 8 (00, 03, 06, 09, 12, 15, 18, 21) を `Promise.allSettled` で並列取得
  - 未来時刻ブロックは 404 となるが許容
- **タイムスタンプ**: `YYYYMMDDhhmmss` (JST)
- **描画**: 純 SVG 折れ線
  - 横軸: 0–24 時、6 時間ごとにグリッド
  - 縦軸: 自動スケール（最小値-1 〜 最大値+1 を整数に丸め、最小幅4°）、目盛は範囲が10°以下なら2°刻み、それ以外5°刻み
  - 折れ線色: `#dc2626` / 折れ線下塗り: `rgba(220,38,38,0.08)`
  - 各データ点に `<title>` 要素でホバー時刻・気温
  - 最新点は強調マーカー（半径 3.2px、白縁取り）
  - 右上に最新値（小数1桁＋時刻）

### 4.5 地点 ↔ パネル連動

地点の選択は以下のいずれかで発火:

- 地図左上の地点ボタンクリック
- 地図上のマーカークリック

選択時の動作 (`selectLocation` 関数):

1. `currentLocation` を更新
2. `map.flyTo` でアニメーション付き移動（duration 0.6s, 各地点の指定ズーム）
3. 該当マーカーのポップアップを開く
4. 地点ボタンの active クラスを切替
5. ヘッダー H1・予報パネル H2 のテキストを当該地点／県名に更新
6. `loadForecast(loc)` 起動（細分タブと気温観測点も連動）
7. `loadTempChart(loc)` 起動

### 4.6 自動更新

| 対象 | 間隔 |
| --- | --- |
| 雨雲ナウキャスト時刻一覧 + タイル切替 | 5 分 |
| 当日気温（AMEDAS） | 10 分 |
| 7 日間予報 | 30 分 |

すべて `currentLocation` を引数に渡して再フェッチ。

### 4.7 競合制御（race protection）

`loadForecast` および `loadTempChart` は非同期処理中に地点が切り替わった場合、応答到着時点で `loc !== currentLocation` を検出してその応答結果を破棄する。これにより遅延応答が後続選択を上書きするのを防ぐ。

### 4.8 雨雲アニメーション

- スライダー: フレームインデックスを直接操作
- 再生ボタン: 500ms 間隔でフレームを進める `setInterval`、末尾で先頭にループ
- バッジ: `実況`（青）／`予報`（赤）でフレーム種別を表示

## 5. UI 仕様

### 5.1 レイアウト

- ヘッダー (`<header>`): 黒地（`#0f172a`）、タイトル ＋ 雨雲取得時刻メタ
- メイン (`<main>`): 2 カラム CSS Grid (`1.4fr : 1fr`)、min-width 900px
  - 左: 地図パネル（地図 ＋ 凡例 ＋ コントロール）
  - 右: 予報パネル（見出し ＋ 気温グラフ ＋ 細分タブ ＋ 7日間リスト）
- フッター: 出典表記
- 900px 未満: 単列、地図 60vh、予報は下に展開

### 5.2 地図パネル

- ベース: 国土地理院 淡色地図
- オーバーレイ: 雨雲ナウキャスト（不透明度 0.7）
- フローティング要素:
  - 地点ボタン群（左上、Leaflet ズーム UI を避ける位置 `left: 50px`）
  - 凡例（左下、降水強度カラーキー）

### 5.3 予報パネル

- 見出し（県名）＋ 発表気象台/発表時刻
- 気温チャート: 高さ 140px、現在値バッジを右上に配置
- 細分タブ: 横並びピル型、active は青塗り
- 日次リスト: 70px (日付) / 56px (アイコン) / 1fr (テキスト+降水確率) / 110px (気温) の 4 列 grid

## 6. データ取得とエラー処理

| 状況 | 振る舞い |
| --- | --- |
| ナウキャスト時刻一覧 fetch 失敗 | ヘッダーメタを「雨雲データの取得に失敗しました」に更新、コンソールにエラー出力 |
| 予報 JSON fetch 失敗 | 予報パネルにエラーメッセージ表示 |
| AMEDAS 観測点表 fetch 失敗 | 気温チャート部に「アメダス観測点表の取得に失敗」表示 |
| AMEDAS 気温が空 | 5 候補まで再試行、それでも無ければ「本日の気温データを取得できませんでした」 |
| `telops.json` fetch 失敗 | 内蔵フォールバック表を使用（主要コードのみ）|
| ナウキャスト個別タイルが 404 | Leaflet 標準動作（透過）。明示エラー表示なし |

## 7. CORS

`www.jma.go.jp/bosai/...` および `cyberjapandata.gsi.go.jp/xyz/...` のいずれもクロスオリジン許可済み。プロキシ等は不要。

## 8. 既知の制限

- 雨雲ナウキャストの最大ネイティブズームは 10。それ以上ズームインするとブラウザ側でアップサンプリングされる。
- 県内一次細分の名称は JMA 側の表記変更があり得る。`subAreaMatch` の部分一致でカバーするが、完全に名称変更された場合フォールバック（index 0）に切り替わるためソース修正が望ましい。
- AMEDAS 観測点は最寄り順で気温観測のあるものを採用する。複数候補がほぼ等距離にある場合、結果が日によって入れ替わる可能性がある（地点コード固定なら防げるが、現状はパフォーマンスより堅牢性優先）。
- ローカルファイル (`file://`) で開く場合、ブラウザの fetch 制約により JMA 側の CORS 判定が異なるケースがある。HTTP/HTTPS 経由（GitHub Pages など）での起動を推奨。
